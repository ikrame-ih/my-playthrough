Arquitectura PERN y flujo de datos

Tres capas apiladas (presentación, negocio, persistencia) y leyenda numerada 1–8.

Pasos del flujo (ejemplo guardar / consultar)

  1. Acción del usuario en la interfaz
  2. Petición HTTP (JSON) hacia la API
  3. Validación JWT y políticas (middleware)
  4. Ejecución de la lógica en controladores
  5. Consulta SQL parametrizada al motor
  6. Filas devueltas al controlador
  7. Respuesta JSON al cliente
  8. React actualiza estado y vista

Ejemplo de ruta: PUT /api/games/:id o POST /api/games con JWT.

Los nombres de tablas en el pie del diagrama son una leyenda resumida; el esquema desplegable está en docs/sql/schema.sql. Para PDF: imprimir desde el navegador.
