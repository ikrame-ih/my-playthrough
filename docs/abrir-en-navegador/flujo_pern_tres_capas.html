PERN architecture and data flow

Three stacked layers (presentation, business, persistence) and a numbered legend 1–8.

Flow steps (example save / query)

  1. User action in the interface
  2. HTTP request (JSON) to the API
  3. JWT validation and policies (middleware)
  4. Logic execution in controllers
  5. Parameterized SQL query to the engine
  6. Rows returned to the controller
  7. JSON response to the client
  8. React updates state and view

Example route: PUT /api/games/:id or POST /api/games with JWT.

Table names in the diagram footer are a short legend; the full schema is in docs/sql/schema.sql. For PDF: print from the browser.
