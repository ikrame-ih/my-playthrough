Modelo lógico — base de datos

Lo mismo que docs/sql/schema.sql, en página. Tres dibujos para que las flechas no se amontonen; abajo, la tabla de relaciones y el listado de columnas cuentan el detalle (incluida la cardinalidad en palabras, sin números encima del SVG).

Clave primaria (PK) Clave foránea (FK): enlaza con otra tabla Las flechas marcan dependencias; 1:N y casos raros están en la tabla de relaciones.

1. Núcleo de la aplicación (quién es el dueño y de dónde sale el título oficial)

Un usuario puede tener muchas fichas en juegos; una fila de catalogo_juegos puede enlazar muchas fichas cuando llevan catalogo_id.

2. Discusión en una ficha (comentarios enlazan a juego y autor; votos enlazan a comentario y votante)

Cada comentario pertenece a un juego y a un autor; cada voto une un comentario con quien vota.

3. Red social (seguidos, recomendaciones, buscar grupo / LFG)

Cada tabla de la derecha guarda filas que enlazan usuarios y, cuando aplica, un juego (por ejemplo una recomendación o un LFG asociado a una ficha).

Relaciones (clave foránea → tabla destino) y cardinalidad

La columna Cardinalidad lo dice en una frase (un usuario, muchas fichas…). Donde pone M:N, la relación entre usuarios pasa por una tabla intermedia (cada fila es un par seguidor→seguido).

Tabla (hijo) Columna FK Apunta a (padre) Cardinalidad
juegos usuario_id usuarios(id) 1 usuario : N juegos
juegos catalogo_id catalogo_juegos(id) 1 entrada catálogo : N juegos (FK opcional)
juego_comentarios juego_id juegos(id) 1 juego : N comentarios
juego_comentarios usuario_id usuarios(id) 1 usuario : N comentarios
juego_comentarios parent_id juego_comentarios(id) (respuesta en hilo) 1 comentario raíz : N respuestas
juego_comentario_votos comentario_id juego_comentarios(id) 1 comentario : N votos (usuarios distintos)
juego_comentario_votos usuario_id usuarios(id) 1 usuario : N votos (en comentarios distintos)
usuario_seguimientos seguidor_id, seguido_id usuarios(id) (dos veces) M:N usuarios vía filas; cada fila es un par seguidor→seguido
juego_recomendaciones remitente_id, destinatario_id usuarios(id) 1 remitente o destinatario : N filas de recomendación
juego_recomendaciones juego_id juegos(id) (ficha del remitente) 1 juego (ficha remitente) : N envíos a distintos destinatarios
lfg_publicaciones usuario_id usuarios(id) 1 usuario : N publicaciones LFG
lfg_publicaciones juego_id juegos(id) 1 juego : N publicaciones LFG

estado VARCHAR(50) NOT NULL
