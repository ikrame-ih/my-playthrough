MYPLAYTHROUGH — defense crib sheet
Personal notes for oral Q&A. Not a spec — check the repo if something drifted.

---

What it is
SPA PERN: personal game library + light community (follow, recommend to followers only, LFG tied to your own games, threaded comments with votes, public profiles, admin panel). Built as the 2025/26 Web Application Development final project.

Stack
PostgreSQL, Express 5, React 18 + Vite, Node. presentation/ is React + TS + Framer Motion for the defense deck only.

Frontend (client/)
Routes: collection, add/edit game, community, /user/:id, /settings, /recommendations, /search, /game/:id/discussion, /admin. apiFetch adds JSON + Bearer; 401 clears token and reloads. Skeletons, grid/list, sort, flash after save (location.state), react-joyride after register, skip link + main focus. Covers via server proxy (Steam + RAWG).

Backend (server/)
index.js: CORS, json 50kb, routers under /api/*. JWT in auth.middleware; admin role re-read from DB every time. bcrypt cost 10; parameterized SQL. Rate limit ~40/15min on login+register. Cover proxy uses host allowlist.

Database — eight tables
usuarios, catalogo_juegos (shared title/cover), juegos (per user), juego_comentarios (parent_id tree), juego_comentario_votos, usuario_seguimientos, juego_recomendaciones, lfg_publicaciones. CASCADE on user delete.

Why catalogo_juegos
Same title from RAWG/Steam → one catalog row, many juegos rows with different status/scores/hours. Example: three users with Cyberpunk share one catalogo row.

Auth
Register with strong password policy (client + server). Login by email or nombre_usuario. JWT 7d in token; admin still checked in Postgres. PATCH /api/auth/me: nombre_usuario, avatar_id, notificaciones_sonido.

Social rules
Can only recommend to users you follow. LFG must reference a game in your library. Activity feed = comments + LFG from people you follow.

Covers
Client calls /api/games/cover-search (auth). Images load through /api/covers/proxy because Steam/RAWG block direct browser fetches. RAWG_API_KEY optional but improves console coverage. catalogo_id stored when user picks a search result.

Security talking points
JWT + bcrypt, CORS_ORIGIN, SQL params, rate limit, proxy allowlist, admin from DB not token alone, /api/test-db 404 in production.

Admin
List/delete users (modal: type exact username), delete any game, list LFG. Comment moderation in discussion view (author, game owner, or admin).

Demo

Local dev

Tests
58 manual (plan_pruebas.html) + 27 Vitest (normalize, gameLabels, coverUrl, passwordPolicy). V-01–V-04 align with normalize.js.

Common tribunal questions
- Why not Mongo? Relational data, FKs, CASCADE, shared catalog.
- Why JWT? Stateless API, SPA-friendly.
- Why proxy? CORS / hotlink blocks on CDN images.
- Token says admin but 403? Role changed after token issued — we re-query.
- Username unique? lower(trim()) index.

API map (see server/routes/*.js)
auth, games CRUD + comments, users list + public collection, community search/stats/public game, social follows/recs/lfg/activity/unread, admin, covers.

Presentation
presentation/ → GitHub Pages defense deck. PrintDeck.tsx is the static PDF variant (no motion).

Env (never commit)
DB_* or DATABASE_URL, JWT_SECRET, CORS_ORIGIN, RAWG_API_KEY, PORT, VITE_API_URL.

Debugging flow
Browser Network tab → Express log → Postgres constraint code (23505 unique, etc.).
