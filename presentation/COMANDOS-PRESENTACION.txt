=== ORDEN EXACTO (PowerShell) — MyPlaythrough + presentación ===

Abre TRES ventanas de terminal. Respeta el orden 1 → 2 → 3.
Espera a que cada paso arranque sin error antes del siguiente (sobre todo el 1).

--- Terminal 1 (API + base de datos con Docker) ---

cd c:\Users\tizza\MyPlaythrough
docker compose up --build

(Deja esta ventana abierta. Cuando veas que el servicio API escucha, sigue.)

--- Terminal 2 (aplicación web MyPlaythrough — puerto 5173 fijo) ---

cd c:\Users\tizza\MyPlaythrough\client
npm run dev

--- Terminal 3 (diapositivas — puerto 5174 fijo) ---

cd c:\Users\tizza\MyPlaythrough\presentation
npm run dev

--- Navegador ---

Presentación (hablas desde aquí):  http://localhost:5174
Aplicación (demo):                 http://localhost:5173

Los botones de la slide "Demo en vivo" abren la app en otra pestaña SOLO si Terminal 1 y 2 están bien.

=== Si NO usas Docker ===

Terminal 1: Postgres tuyo ya arrancado + esquema aplicado (README).
Luego en server:

cd c:\Users\tizza\MyPlaythrough\server
npm run dev

Después Terminal 2 y 3 igual que arriba.
